Design System & Wireframes — Obsidian Theme

The same layout, screens, and component set as the tactical variant, reskinned in Obsidian's visual identity: soft charcoal surfaces, purple accent, rounded cards, Inter type, graph-view styling, purple wikilinks. This is the theme that visually harmonizes with the user's vault. Build it in React + Tailwind + Vite.

Obsidian dark · purple accent Inter + JetBrains Mono Soft 8px radius Flat & calm Wireframe — layout & language, not final copy
01

Design Tokens

CSS variables are the contract; map them to Tailwind theme tokens. Obsidian's identity is charcoal + purple. One important remap from the tactical theme: since purple is now the brand/primary (used for links, active states, primary actions — just like Obsidian's wikilinks), the AI-output marker moves to cyan so it stays visually distinct from the brand color. Color still carries meaning: purple = primary/links, green = live/healthy, amber = attention, red = cold/destructive, blue = info, cyan = AI.

Design Principles (carry into every screen)

Calm, not loud

Obsidian's restraint: flat surfaces, soft radius, generous breathing room. Information-dense but never noisy.

Color = meaning

Purple primary/links · green live · amber attention · red cold · blue data · cyan = AI only. Never decorative.

Enforce the hard parts

UI gates recall before reveal, surfaces cold domains, refuses to mark things "learned" for free. The friction is the feature.

AI is visibly AI

Generated content is cyan-tagged and grounded with citations. The user always knows what the machine produced.

Feels like the vault

Purple wikilink accent, graph-view nodes, callouts, tag pills — the app visually continues Obsidian rather than clashing with it.

Local & private by default

Lockdown state is glanceable. Local = green, remote = amber. Privacy posture is never hidden.
